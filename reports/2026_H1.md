- 업무 상세:
  - **고객 테스트**
  - 1. 기존 제공받은 CMM Script 상에서 “FLASHFILE.TARGET” 선언 시 “/dualport” 옵션 제거
  - 2. “FLASHFILE.GETID”를 통한 JEDEC ID 값 확인 성공
  - 3. “FLASHFILE.ERASE 0x0++0xFFFF” 명령어 실행
  - 4. “FLASHFILE.DUMP 0x0 /Long”을 통한 값 확인 시, 기존 값 유지함, FLASHFILE.ERASE가 안됨
  - 5. CPU FW를 통해 ERASE 진행, 해당 영역은 0xFFFF_FFFF로 바뀜, CPU FW를 통한 ERASE는 정상 동작함
  - 6. “FLASHFILE.LOAD * 0x0” 명령어를 통해 *.bin 파일 로드 진행
  - 7. “FLASHFILE.DUMP 0x0” 명령어를 통해 *.bin 값과 동일함을 확인, FLASHFILE.LOAD 성공, 정상 동작함
  - **정리**
  - flashfile.erase의 명령어가 정상동작되지 않음
  - CPU FW를 통한 erase는 정상동작
  - 명령어에 대한 오류를 본사측에서 확인 요청
