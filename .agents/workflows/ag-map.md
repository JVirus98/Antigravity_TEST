# /ag-map - 프로젝트 맵 갱신

이 워크플로우는 현재 실제 파일 구조를 스캔하여 `project-map.md`를 최신 상태로 유지합니다.

## 실행 단계

1. **실제 디렉토리 구조 스캔**
   - `c:\TEST\dailyreport\` 전체 파일 목록을 확인합니다.
   - 신규 파일, 삭제된 파일, 변경된 파일을 식별합니다.

2. **reports/ 상태 확인**
   - 각 반기 파일(`YYYY_H?.md`)의 크기와 보고서 건수를 확인합니다.
   - `_index.md`와 실제 파일 목록의 일치 여부를 검증합니다.

3. **스크립트 변경 사항 반영**
   - 각 .py 파일의 역할 설명이 최신인지 확인합니다.
   - 새로 추가되거나 삭제된 스크립트를 반영합니다.

4. **기술 부채 섹션 업데이트**
   - 현재 알려진 이슈 및 TODO 항목을 기록합니다.
   - 완료된 작업 항목은 체크 표시합니다.

5. **project-map.md 갱신**
   - 마지막 업데이트 날짜를 오늘로 변경합니다.
   - 변경된 내용을 반영하여 파일을 덮어씁니다.
